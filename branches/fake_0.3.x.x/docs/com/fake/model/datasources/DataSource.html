public function call(correlationId:String, service:String, args:Object = null):void

Parameters

executeCall()method 
protected function executeCall():void

A call is made with the service and args pulled from the object at the top of the call queue. onSuccess and onFail are used in the responder.

onSuccess()method 
protected function onSuccess(result:Object):void

A FakeEvent gets dispatched with the data parameter containing the result of the call. The type parameter is filled with the service string from the object at the top of the call queue. The object is then shifted off of the queue. The call queue is checked for any calls pending. If there are calls pending excecuteCall is called. It is called reDundAnt()

result:Object — data object returned from the net call
